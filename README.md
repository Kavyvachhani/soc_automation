# 🛡️ Attest — SOC 2 Compliance Evidence Automation

**Employee onboarding evidence pipeline** with automated NDA generation, e-signature capture,
approval-gated provisioning, and full audit trails. Runs end-to-end locally in mock mode (no
cloud needed) or deploys to AWS via Terraform + GitHub Actions.

---

## Architecture

```
┌────────────────────┐
│   Streamlit Portal │   Employee uploads offer letter, reviews & signs NDA
│   (portal/app.py)  │
└────────┬───────────┘
         │ PDF upload
         ▼
┌────────────────────┐
│  Offer Processor   │   Extracts employee data (AI + regex fallback)
│  (Lambda / inline) │   Auto-fills NDA template → unsigned NDA PDF
└────────┬───────────┘
         │ employee.json + nda-unsigned.pdf
         ▼
┌────────────────────┐
│  Portal: Sign NDA  │   Consent checkbox → typed signature → audit trail
│  e-signature       │   {timestamp, IP, user-agent, doc hash before/after}
└────────┬───────────┘
         │ signed-nda.pdf + nda-audit-trail.json
         ▼
┌────────────────────┐
│  Signed Processor  │   Verifies audit trail → repository_dispatch to GitHub
│  (Lambda / inline) │
└────────┬───────────┘
         │ nda-signed event
         ▼
┌────────────────────┐
│  GitHub Actions    │   environment: provisioning (REQUIRED REVIEWER GATE)
│  provisioning.yml  │   Tech Lead must click "Approve" before any provisioning
└────────┬───────────┘
         │ approved
         ▼
┌────────────────────┐
│  provision.py      │   Resolves role from catalog.yaml → MOCK provisions
│  (scripts/)        │   Writes access-granted.csv + evidence-index.json
└────────────────────┘
```

### Modes

| Mode | Storage | Processing | Approval Gate |
|------|---------|------------|---------------|
| **MOCK** (`MOCK_MODE=true`) | Local filesystem (`./data/`) | Inline in portal | "Approve" button in UI |
| **LIVE** (`MOCK_MODE=false`) | S3 vault (KMS-encrypted) | AWS Lambda (S3 triggers) | GitHub Environment reviewer |

## Run Locally in Mock Mode (No Cloud)

### Prerequisites

- **Python 3.10+** (3.12 recommended)
- **pip**

### Step 1 — Install Dependencies

```bash
cd attest

# Create a virtual environment (recommended)
python3 -m venv .venv
source .venv/bin/activate

# Install portal + script dependencies
pip install -r portal/requirements.txt
```

### Step 2 — Generate Sample Data

```bash
python scripts/seed_data.py
```

This creates `sample_data/offer-letter.pdf` and several signed policy acknowledgement PDFs.

### Step 3 — Launch the Portal

```bash
MOCK_MODE=true streamlit run portal/app.py
```

The portal opens at **http://localhost:8501**.

### Step 4 — End-to-End Test

1. **Upload**: Click "Download sample offer letter" → upload the downloaded PDF → click "Process →"
2. **Review**: The system extracts employee data and shows the auto-generated NDA.
3. **Sign**: Check the consent box, type the full legal name (e.g. "Priya Sharma"), click "Sign NDA & Submit".
4. **Approve**: Enter an approver name (or use the default), click "Approve & Provision Access".
5. **Done**: View the evidence index, access grants table, and download all evidence files.

### Verify Evidence Files

After completing the flow, check the generated evidence:

```bash
ls -la data/employees/EMP-*/

# Expected files:
#   offer-letter.pdf          ← uploaded offer letter
#   employee.json             ← extracted employee data
#   nda-content.txt           ← plain text NDA
#   nda-unsigned.pdf          ← auto-generated unsigned NDA
#   signed-nda.pdf            ← signed NDA with embedded signature
#   nda-audit-trail.json      ← full e-signature audit trail
#   access-granted.csv        ← provisioned access grants
#   evidence-index.json       ← complete evidence index
```

## Deploy to AWS

### Prerequisites

- **Terraform ≥ 1.6**
- **AWS CLI** configured with appropriate permissions
- A GitHub repository for this code
- A GitHub **Personal Access Token** (for `repository_dispatch`)

### Step 1 — Terraform Apply

```bash
cd terraform

terraform init

terraform apply \
  -var="evidence_bucket_name=my-attest-vault-UNIQUE" \
  -var="github_org=my-github-org" \
  -var="github_repo=attest" \
  -var="anthropic_api_key=sk-ant-..." \
  -var="github_token=ghp_..."
```

> **Tip**: Use a `terraform.tfvars` file or environment variables for secrets:
> ```bash
> export TF_VAR_anthropic_api_key="sk-ant-..."
> export TF_VAR_github_token="ghp_..."
> ```

### Step 2 — Configure GitHub Repository

1. **Secrets** (Settings → Secrets and variables → Actions):
   - `AWS_ROLE_ARN` — from Terraform output `github_actions_role_arn`
   - `AWS_REGION` — e.g. `us-east-1`
   - `S3_BUCKET` — from Terraform output `evidence_bucket_name`

2. **Environment** (Settings → Environments):
   - Create an environment named `provisioning`
   - Add at least one **required reviewer** (the tech lead)

### Step 3 — Deploy Lambda Code

Push to `main` with changes in `lambda/` — the `deploy-lambdas.yml` workflow will
automatically package and deploy both Lambda functions via OIDC.

Or trigger manually: Actions → Deploy Lambda Functions → Run workflow.

### Step 4 — Seed the Vault

```bash
python scripts/seed_data.py --mode upload --bucket my-attest-vault-UNIQUE
```

### Step 5 — Run Live

```bash
MOCK_MODE=false S3_BUCKET=my-attest-vault-UNIQUE streamlit run portal/app.py
```

## Security & Compliance

### Non-Negotiable Practices

| Practice | Implementation |
|----------|----------------|
| **No long-lived AWS keys** | GitHub Actions uses OIDC → STS; trust scoped to `repo:<org>/<repo>:*` |
| **Approve before provision** | GitHub `provisioning` Environment with required reviewers |
| **No plaintext passwords** | Provisioning is mock; secrets via env vars / Secrets Manager |
| **S3 suffix filters** | `offer-letter.pdf` and `signed-nda.pdf` suffixes prevent infinite loops |
| **AI graceful degradation** | Anthropic API failure falls back to regex heuristics; pipeline never hard-fails |
| **Encrypted evidence vault** | S3 with KMS (AES-256), versioning, public access blocked, TLS-only |
| **Full audit trail** | Server timestamp, IP, user-agent, consent flag, signature method, doc hash before+after |
| **WORM support** | Optional Object Lock in GOVERNANCE mode (`enable_worm = true`) |

### E-Signature Audit Trail

Every signed NDA is accompanied by `nda-audit-trail.json`:

```json
{
  "emp_id": "EMP-ABCD1234",
  "signer_name": "Priya Sharma",
  "timestamp_utc": "2026-06-10T00:00:00.000000Z",
  "source_ip": "127.0.0.1",
  "user_agent": "Attest-Streamlit-Portal/1.0",
  "consent": true,
  "signature_method": "typed-name",
  "document_hash_before": "sha256:...",
  "document_hash_after": "sha256:..."
}
```

## Terraform Variables

| Variable | Type | Default | Description |
|----------|------|---------|-------------|
| `aws_region` | string | `us-east-1` | AWS region |
| `project_name` | string | `attest` | Resource name prefix |
| `evidence_bucket_name` | string | *required* | Globally-unique S3 bucket name |
| `github_org` | string | *required* | GitHub organisation |
| `github_repo` | string | `attest` | GitHub repo name |
| `enable_provisioning` | bool | `false` | Attach scoped IAM provisioning policy |
| `enable_worm` | bool | `false` | Enable S3 Object Lock (GOVERNANCE) |
| `anthropic_api_key` | string (sensitive) | `""` | For AI extraction in offer_processor |
| `github_token` | string (sensitive) | `""` | For repository_dispatch in signed_processor |
| `readonly_policy_arn` | string | `AmazonS3ReadOnlyAccess` | Policy for fresher role |
| `developer_policy_arn` | string | `PowerUserAccess` | Policy for experienced role |

## Development

### Adding a New Policy

1. Add the markdown policy to `policies/`.
2. Update `seed_data.py` to generate an acknowledgement PDF.
3. The portal automatically picks up policies from the `POLICIES_DIR`.

### Switching to Real Provisioning

1. Set `enable_provisioning = true` in Terraform.
2. Modify `scripts/provision.py` to make real IAM calls (the `mock=true` flag
   and print statements show you where to add real logic).
3. **Never** email plaintext passwords — use SSO invite links or temporary credentials.
